CRUD

    ¿Qué es un CRUD?

    CRUD es un acrónimo que representa las cuatro operaciones básicas que se pueden realizar en una aplicación que maneja datos:

    • Crear (Create): Permite agregar nuevos datos a la base de datos o sistema de almacenamiento.
    • Leer (Read): Permite obtener datos existentes de la base de datos o sistema de almacenamiento.
    • Actualizar (Update): Permite modificar los datos existentes en la base de datos o sistema de almacenamiento.
    • Eliminar (Delete): Permite eliminar datos de la base de datos o sistema de almacenamiento.

    ¿Cómo funciona este CRUD?

    1. Agregar una tarea: En el campo de entrada de texto, ingresa el texto de una nueva tarea y haz clic en el botón "Agregar". La función `addTask()` se encarga de enviar una solicitud POST al servidor para agregar la nueva tarea. Luego, la función `renderTask()` muestra la tarea agregada en la lista.
    2. Editar una tarea: Para editar una tarea existente, haz clic en el botón "Editar" de la tarea que deseas modificar. Se te pedirá un nuevo texto para la tarea. Si proporcionas un nuevo texto y confirmas la edición, la función `editTask()` enviará una solicitud PUT al servidor para actualizar la tarea. Luego, se recargará la lista de tareas para mostrar la tarea actualizada.
    3. Eliminar una tarea: Para eliminar una tarea, haz clic en el botón "Eliminar" de la tarea que deseas eliminar. La función `deleteTask()` enviará una solicitud DELETE al servidor para eliminar la tarea. Luego, se eliminará la tarea de la lista.

    Explicación del Server (server.js)

    El archivo server.js es el componente del lado del servidor que maneja las solicitudes y respuestas HTTP relacionadas con las operaciones CRUD de tareas. Utiliza el framework web Express.js para simplificar el manejo de rutas y middlewares.

    Explicación del Script (script.js)

    El archivo script.js es el componente del lado del cliente que se ejecuta en el navegador web. Es responsable de manejar la interacción del usuario con la interfaz de la página web y realizar las solicitudes HTTP al servidor para realizar las operaciones CRUD de tareas.

    Explicación de la Database (database.js)

    El archivo database.js es el componente que se encarga de la conexión y las operaciones con la base de datos SQLite3. Se utiliza para almacenar y recuperar las tareas en el servidor.
